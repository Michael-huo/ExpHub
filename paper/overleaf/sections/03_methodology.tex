\section{Methodology}
\label{sec:methodology}
% ==================================================

\subsection{Framework Overview and Problem Formulation}
\label{subsec:framework_formulation}

\HVM{} formulates edge--cloud VSLAM transmission as a VSLAM-oriented vision--language memory coding problem. Instead of sending a dense image stream to the cloud, the edge device selects sparse boundary frames and summarizes the omitted intervals with compact motion memory, semantic anchors, and language memory. The cloud then uses a vision--language conditioned world-model decoder to reconstruct task-usable intermediate frames, and the resulting sequence is consumed by a conventional cloud-side VSLAM estimator.

\begin{figure*}[!t]
\centering
\includegraphics[width=\textwidth]{figures/fig2.png}
\caption{Overall framework of HVM-Codec for edge--cloud VSLAM visual transmission. The edge-side encoder converts dense visual streams into boundary frames, motion memory, semantic anchors, and compact language memory, while the cloud-side world-model decoder reconstructs task-usable visual sequences for downstream VSLAM. The same memory format is used for inference and memory-guided refinement.}
\label{fig:framework}
\end{figure*}

Fig.~\ref{fig:framework} gives the global pipeline. The edge device performs lightweight memory encoding and transmits only the sparse vision--language memory. The cloud receives this memory, reconstructs the missing visual continuity, and runs VSLAM on the reconstructed sequence. The same encoder is also used offline to prepare memory-guided refinement pairs, so the memory format seen during training is consistent with the memory format used during inference.

Let the dense visual stream captured by the edge device be
\begin{equation}
\Iseq=\{I_1,I_2,\ldots,I_T\},
\label{eq:input_stream}
\end{equation}
where \(I_t\) denotes the \(t\)-th image. The encoder partitions the stream into \(N\) vision--language memory units,
\begin{equation}
\mathcal{U}=\{U_1,\ldots,U_N\},\quad
U_i=\{I_{b_{i-1}},I_{b_i},S_i,A_i,L_i,L_i^{-}\},
\label{eq:memory_unit}
\end{equation}
where \(b_{i-1}\) and \(b_i\) are neighboring event boundaries. The frames \(I_{b_{i-1}}\) and \(I_{b_i}\) are transmitted as boundary frames. The variable \(S_i\) denotes motion memory, \(A_i\) denotes semantic anchors, and \(L_i\) and \(L_i^{-}\) denote positive and negative language memory, respectively.

Each component in \(U_i\) has a distinct role in task-usable visual reconstruction. Boundary frames preserve visual and geometric anchors at the ends of a unit, including appearance, perspective, and structural cues that cannot be accurately described by language alone. Motion memory describes the coarse viewpoint evolution inside the unit, especially forward motion, turns, and mixed transitions that affect intermediate camera poses. Semantic anchors capture scene/content changes that should constrain reconstruction, such as the emergence of distinctive scene structures. Language memory then replaces part of the omitted intermediate visual detail with compact semantic conditions, providing the decoder with motion and scene priors without transmitting every frame.

The transmitted memory representation is written as
\begin{equation}
\Mmem=\{\mathcal{U},\Vocab,\mathrm{metadata}\},
\label{eq:transmitted_memory}
\end{equation}
where \(\Vocab\) contains the language vocabulary and reconstruction conditions shared by the encoder and decoder. The metadata records temporal ordering, unit length, and boundary-frame indices needed to merge reconstructed units. Unlike a conventional bitstream, \(\Mmem\) does not contain all pixel samples of the omitted frames; it contains the sparse vision--language conditions needed by the cloud decoder.

The communication budget can be described as
\begin{equation}
\begin{aligned}
B(\Mmem)=&B(\Kframes)+B(\Smotion)+B(\Aanchor)+B(\Llang)\\
&+B(\Vocab)+B(\mathrm{metadata}),
\end{aligned}
\label{eq:budget}
\end{equation}
where \(\Kframes\) are the transmitted boundary frames, \(\Smotion\) is the motion memory, \(\Aanchor\) is the set of semantic anchors, and \(\Llang\) is the language memory. This expression is used to reason about transmission reduction; it is not a claim that all components contribute equally to the payload.

This formulation differs from ordinary sparse keyframe transmission. A keyframe-only scheme transmits selected frames but leaves the omitted intervals unavailable unless an additional interpolation or reconstruction process is introduced. In \HVM{}, the transmitted representation includes sparse boundary frames together with motion memory, semantic anchors, and language memory, and the cloud-side world-model decoder reconstructs the omitted intervals under these vision--language conditions. This design can support sparser boundary-frame transmission than a pure keyframe-only representation, because the missing intervals are not simply discarded; they are reconstructed as task-usable visual content for downstream VSLAM evaluation.

The task-level design criterion is therefore expressed conceptually as
\begin{equation}
\begin{aligned}
B(\Mmem) &\ll B(\Iseq),\\
\mathrm{APE}_{\mathrm{RMSE}}(\hat{\mathbf{T}},\mathbf{T}) &\leq \epsilon_{\mathrm{task}},
\end{aligned}
\label{eq:objective}
\end{equation}
where \(\hat{\mathbf{T}}\) is the trajectory estimated by the VSLAM estimator from the reconstructed sequence and \(\mathbf{T}\) is the reference trajectory when available. This criterion states the intended tradeoff: the memory should substantially reduce communication while preserving downstream VSLAM trajectory quality. In this paper, APE RMSE is the primary reported trajectory metric, while transmission reduction, edge-side motion-estimation cost, and auxiliary image-fidelity metrics are reported separately in the experiments.

\subsection{Motion--Semantic Event Memory Encoding}
\label{subsec:motion_semantic_segmentation}

The edge encoder converts a dense frame stream into vision--language memory units. A useful unit boundary should capture either a meaningful camera-motion transition or a scene-content change that affects later reconstruction. HVM-Codec therefore uses two complementary event cues: a motion trend response and a semantic anchor response. The motion trend response captures viewpoint evolution and motion transitions. Turning, start/stop behavior, and mixed motion can make reconstruction harder because intermediate views may involve larger viewpoint changes and weaker common-view overlap. The semantic anchor response complements this cue by capturing embedding-level scene/content changes that occur even when the coarse motion state remains stable.

\begin{figure*}[!t]
\centering
\includegraphics[width=0.92\textwidth]{figures/fig3.png}
\caption{Motion--semantic response fusion. Motion trend responses and semantic anchor responses are fused to divide the dense visual stream into vision--language memory units. Each unit is represented by boundary frames, motion memory, semantic anchors, and compact language memory.}
\label{fig:event_segmentation}
\end{figure*}

As shown in Fig.~\ref{fig:event_segmentation}, the encoder first estimates coarse motion evidence between consecutive frames. Phase correlation is used because it is inexpensive and produces a global translational response suitable for language-level motion memory. It is not intended to replace dense optical flow or precise feature tracking; rather, it provides a low-cost indication of whether the camera is moving forward, turning, or undergoing a mixed transition. For frame pair \((I_{t-1},I_t)\), the response is
\begin{equation}
(\Delta x_t,\Delta y_t,\rho_t)=\operatorname{PC}(I_{t-1},I_t),\quad
s_t=\Gamma_m(\Delta x_t,\Delta y_t,\rho_t),
\label{eq:motion_state}
\end{equation}
where \(\Delta x_t\) and \(\Delta y_t\) denote the dominant translational response, \(\rho_t\) is the response confidence, and \(s_t\in\{\mathrm{forward},\mathrm{left\mbox{-}turn},\mathrm{right\mbox{-}turn},\mathrm{mixed}\}\) is the resulting coarse motion state.

For a candidate interval \([t_0,t]\), the accumulated motion response is
\begin{equation}
R_m(t_0,t)=\sum_{k=t_0+1}^{t}\phi_m(s_{k-1},s_k,\rho_k),
\label{eq:motion_event_score}
\end{equation}
where \(\phi_m(\cdot)\) measures confidence-weighted changes in the coarse motion state. This response tends to increase around turns, starts, stops, and mixed-motion transitions, which are important for reconstructing viewpoint evolution.

Motion-triggered boundaries are especially useful around turns and mixed-motion intervals. In these segments, the world-model decoder may need to infer intermediate views across larger viewpoint changes, weaker co-visible regions, and stronger geometric uncertainty. Shortening such reconstruction units helps provide clearer motion memory and reduces the distance between the transmitted boundary frames that constrain the unit.

Motion memory alone is insufficient for VSLAM-oriented transmission because scene appearance may change while the motion state remains constant. The encoder therefore measures semantic anchor response in an embedding space:
\begin{equation}
\begin{aligned}
d_t &= 1-\cos(f(I_t),f(I_{t-1})),\\
R_v(t_0,t) &= \sum_{k=t_0+1}^{t} d_k,
\end{aligned}
\label{eq:visual_novelty}
\end{equation}
where \(f(\cdot)\) is an image encoder, \(d_t\) is a frame-level semantic-anchor response, and \(R_v(t_0,t)\) accumulates this response over the candidate interval. Semantic anchors denote embedding-level scene/content changes rather than manually annotated object categories. This response helps capture intervals containing new landmarks, distinctive textures, or visual changes that may provide trackable structure after reconstruction.

The semantic anchor response is complementary to motion memory because a long straight corridor or forward-motion segment may remain in one coarse motion state while the observed scene changes. In such cases, the boundary frames may become far apart, and the omitted interval may include doors, signs, people, landmarks, corridor-structure changes, or object appearance/disappearance that is not indicated by motion alone. The semantic anchor response can indicate these appearance/content changes and provide extra constraints for language memory and reconstruction. As engineering intuition, route memory often combines motion instructions such as going straight or turning with landmarks encountered along the path.

The event boundary set is selected by fusing the motion trend and semantic anchor responses:
\begin{equation}
\mathcal{B}=\Pi_{\Omega}\left(\left\{t\mid R_m(t_0,t)>\tau_m\ \vee\ R_v(t_0,t)>\tau_v\right\}\right),
\label{eq:event_boundary}
\end{equation}
where \(\tau_m\) and \(\tau_v\) are response thresholds and \(\Pi_{\Omega}(\cdot)\) projects detected boundaries to feasible temporal positions. This projection enforces practical constraints such as minimum unit length and valid boundary-frame availability. Neighboring elements of \(\mathcal{B}\) define the vision--language memory units in (\ref{eq:memory_unit}).

For each unit, the encoder summarizes the interval with motion memory and semantic anchors:
\begin{equation}
\begin{aligned}
S_i &= \operatorname{Merge}\{s_t\mid t\in[b_{i-1},b_i]\},\\
A_i &= \{t\in[b_{i-1},b_i]\mid d_t>\tau_a\ \vee\ R_v(b_{i-1},t)>\tau_v\}.
\end{aligned}
\label{eq:unit_memory_summary}
\end{equation}
The motion memory \(S_i\) records the dominant motion trend and major transitions in the unit. The anchor set \(A_i\) marks frames or intervals as semantic anchors that should influence the language memory and decoding conditions, even when those frames are not transmitted as images.

The final step of the edge encoder is to convert motion and semantic-anchor responses into compact language memory. This language memory is constructed from language templates rather than produced by a heavy online captioning model. A fixed language vocabulary and motion-conditioned language templates map the low-cost measurements into short language conditions:
\begin{equation}
\begin{aligned}
L_i &= L_{\mathrm{base}}\oplus L_{\mathrm{motion}}(S_i)\oplus L_{\mathrm{anchor}}(A_i)\oplus L_{\mathrm{constraint}},\\
L_i^{-} &= L_{\mathrm{blur}}\oplus L_{\mathrm{distortion}}\oplus L_{\mathrm{drift}}\oplus L_{\mathrm{jump}}\oplus L_{\mathrm{hallucination}}.
\end{aligned}
\label{eq:language_memory}
\end{equation}
Here \(L_{\mathrm{base}}\) specifies a first-person navigation-view prior, \(L_{\mathrm{motion}}(S_i)\) describes the coarse motion trend, \(L_{\mathrm{anchor}}(A_i)\) inserts semantic-anchor descriptions when available, and \(L_{\mathrm{constraint}}\) states temporal and geometric reconstruction preferences. The negative language condition \(L_i^{-}\) discourages blur, distortion, temporal drift, sudden viewpoint jumps, and inconsistent hallucinated objects. Because this language memory is encoded from language templates, the edge encoder avoids heavy online generative captioning while still transmitting useful high-level conditions for cloud-side reconstruction.

\subsection{Dual-Branch World-Model Decoding}
\label{subsec:wm_lora_decoding}

The cloud-side decoder has two coupled branches. The inference branch reconstructs task-usable intermediate frames from the transmitted vision--language memory, while the refinement branch performs \emph{memory-guided world-model refinement}. This refinement uses the same vision--language memory format as inference, exposing the decoder during refinement to the same type of boundary frames, motion memory, semantic anchors, and language memory that it receives during deployment. This design keeps the decoder input distribution consistent between offline refinement and online deployment.

\begin{figure}[!t]
\centering
\includegraphics[width=0.95\columnwidth]{figures/fig4.png}
\caption{Dual-branch world-model decoding. The inference branch reconstructs intermediate frames from transmitted boundary frames and language memory. The refinement branch uses the same memory format to construct LoRA-based refinement pairs while keeping the pretrained world model frozen.}
\label{fig:lora_decoder}
\end{figure}

Fig.~\ref{fig:lora_decoder} illustrates the two branches. In the inference branch, each vision--language memory unit is decoded from its two boundary frames and language memory:
\begin{equation}
\hat{\mathcal{I}}_i
=G_{\theta+\Delta\theta}(I_{b_{i-1}},I_{b_i},L_i,L_i^{-}),
\label{eq:wm_decode}
\end{equation}
where \(G_\theta\) is the pretrained world model and \(\Delta\theta\) denotes the LoRA low-rank increments used by the implementation-level WM+LoRA decoder. The boundary frames constrain the start and end appearance of the unit, the positive language memory provides motion and semantic priors, and the negative language memory discourages visual artifacts that are harmful for VSLAM.

The reconstructed units are then temporally merged:
\begin{equation}
\Ihat=\operatorname{Merge}(\hat{\mathcal{I}}_1,\hat{\mathcal{I}}_2,\ldots,\hat{\mathcal{I}}_N;\mathcal{B}),
\label{eq:temporal_merge}
\end{equation}
where \(\mathcal{B}\) preserves the original unit order and shared boundary-frame structure. Shared boundaries prevent neighboring units from being treated as unrelated clips and help maintain temporal continuity before the reconstructed sequence is passed to the VSLAM estimator. The decoder is therefore not a conventional entropy or bitstream decoder. It is a conditional world-model decoder that uses sparse memory and learned visual priors to synthesize plausible intermediate continuity for downstream use.

For each unit \(i\), let \(\mathcal{I}^{\mathrm{mid}}_i=\{I_t\}_{t=b_{i-1}+1}^{b_i-1}\) denote its original intermediate frames. In the refinement branch, the same edge encoder converts raw training sequences into memory-guided refinement pairs:
\begin{equation}
\mathcal{D}_{\mathrm{LoRA}}
=\{(I_{b_{i-1}},I_{b_i},L_i,L_i^{-},\mathcal{I}^{\mathrm{mid}}_i)\}_{i=1}^{N_d}.
\label{eq:lora_dataset}
\end{equation}
The last term \(\mathcal{I}^{\mathrm{mid}}_i\) denotes the original intermediate frames within unit \(i\). Because the pair construction uses the same boundary selection, motion memory, semantic anchors, and language memory as inference, the refinement data use the same type of conditioning that the decoder receives during deployment.

In this implementation, the refinement is realized with Low-Rank Adaptation (LoRA): the pretrained world model is kept frozen, and only lightweight low-rank increments are learned in selected layers. This avoids retraining the full world model and makes limited-resource refinement more feasible. A reconstruction-oriented refinement objective can be written as
\begin{equation}
\begin{aligned}
\Delta\theta^*=\arg\min_{\Delta\theta}\sum_i
&\mathcal{L}_{\mathrm{gen}}\bigl(
G_{\theta+\Delta\theta}(I_{b_{i-1}},I_{b_i},L_i,L_i^{-}),\\
&\mathcal{I}^{\mathrm{mid}}_i
\bigr),
\end{aligned}
\label{eq:lora_finetune}
\end{equation}
where \(\mathcal{L}_{\mathrm{gen}}\) denotes the generation training loss used by the underlying video world model. The purpose of this refinement is not to make the images uniformly more visually pleasing or pixel-identical, but to bias the decoder toward navigation-style visual transitions, viewpoint stability, geometric continuity, and trackable structures. It is intended to reduce VSLAM-harmful artifacts such as hallucinated structures, local geometric distortion, and implausible intermediate viewpoints. The reconstructed frames remain task-usable reconstructions whose value is assessed primarily through downstream VSLAM trajectory quality.
