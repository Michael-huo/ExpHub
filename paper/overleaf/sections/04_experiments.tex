\section{Experimental Evaluation}
\label{sec:experiments}
% ==================================================

\subsection{Experimental Setup}
\label{subsec:exp_setup}

The evaluation uses three types of sequences: NCD, TUM, and GDUT. NCD and TUM provide public benchmark sequences, while GDUT contains real robot data collected in our deployment environment. This combination allows the experiments to cover both standard datasets and robot-captured visual streams under the same \VLMemCodec{} evaluation protocol. Fig.~\ref{fig:experimental_platform} shows the edge--cloud hardware platform used for the real-world GDUT experiments.

\begin{figure}[!b]
\centering
\includegraphics[width=\columnwidth]{figures/fig5.png}
\caption{Edge--cloud experimental platform used in the real-world GDUT experiments. The edge platform consists of an AGV equipped with a RealSense D455 camera, an onboard NUC, and a Livox MID-360 LiDAR, while the cloud platform is a multi-GPU server connected through a low-bandwidth sparse transmission setting.}
\label{fig:experimental_platform}
\end{figure}

The edge platform consists of an AGV equipped with a RealSense D455 camera, an onboard NUC, and a Livox MID-360 LiDAR. The cloud platform is a multi-GPU server connected through a sparse transmission setting. In this configuration, the edge device encodes a VLMem payload from raw visual streams, and the cloud performs world-model reconstruction and VSLAM evaluation.

All experiments follow a unified end-to-end evaluation pipeline. The preparation stage organizes image sequences and trajectory references; the encoder converts dense frames into the transmitted VLMem payload; the decoder produces a reconstructed sequence from visual anchors and language cues; and the evaluation stage runs the VSLAM estimator on both original and reconstructed sequences. Unless otherwise stated, phase correlation (PC) is used as the edge-side motion estimator. The cloud uses a vision--language conditioned world-model decoder. We compare the original world-model decoder (WM decoder) with the same decoder refined through MGDR (WM+MGDR decoder). In the ablation, WM denotes the original world-model decoder, while WM+MGDR denotes the world-model decoder refined through MGDR, implemented with LoRA-based low-rank increments.

Trajectory quality is measured from the VSLAM estimator output using evo after Sim(3) alignment, and the reported task metric is APE RMSE. Compression is measured by comparing the size of the original image sequence with the size of the transmitted VLMem payload. LPIPS, SSIM, and FID are used only as auxiliary image-fidelity metrics in the MGDR ablation; they are not the final task-level objective of \VLMemCodec.

\subsection{End-to-End Compression and VSLAM Accuracy}
\label{subsec:compression_accuracy}

The first experiment evaluates end-to-end transmission reduction and VSLAM trajectory accuracy across nine test fragments.

Table~\ref{tab:compression_accuracy} reports that, on average, the raw image sequence size is 403.42 MiB, while the transmitted VLMem payload is 11.06 MiB. This corresponds to an average data reduction of 96.8\%, indicating that the proposed representation substantially reduces transmitted visual data.

The VSLAM results indicate that this reduction can retain task-usable geometric information. ORI and REC denote the original and reconstructed sequences, respectively. Table~\ref{tab:compression_accuracy} still reports absolute APE RMSE, but it now also lists the GT path length \(L_{\mathrm{GT}}\) so each RMSE value can be read with its trajectory scale. The path-normalized deviation \(|\Delta|/L_{\mathrm{GT}}\) helps avoid over-interpreting large absolute RMSE changes on longer trajectories or percentage changes when ORI APE is very small.

The average ORI APE is 0.698 m, while the average REC APE is 0.735 m. \(\Delta\)APE is computed as REC APE minus ORI APE, giving an average signed difference of +0.037 m and an average path-normalized deviation of 0.898\%. These averages should be interpreted together with the sequence-level variation. Different fragments contain different scene texture, motion patterns, reconstruction difficulty, and VSLAM estimator sensitivity, so the same memory-based transmission mechanism can affect feature availability and temporal continuity differently across sequences. Some reconstructed sequences obtain lower APE than their original counterparts, whereas others obtain higher APE. A negative \(\Delta\)APE is not a physical-quality claim about reconstructed frames; it only indicates that the VSLAM estimator estimated a lower aligned trajectory error on that reconstructed sequence. This behavior is expected because \VLMemCodec{} does not aim to reconstruct pixel-identical videos, but to retain enough geometric continuity and trackable texture for downstream VSLAM.

The main interpretation of Table~\ref{tab:compression_accuracy} is therefore the compression--trajectory tradeoff: the transmitted VLMem payload greatly reduces visual data while REC APE remains close to ORI APE on average. This supports the use of Vision--Language Memory for task-usable visual reconstruction under the reported evaluation protocol, rather than claiming frame-level physical fidelity.

Fig.~\ref{fig:trajectory_map} complements Table~\ref{tab:compression_accuracy} by comparing trajectories and point-cloud reconstructions on NCD, TUM, and GDUT. The trajectory and point-cloud similarity in the shown examples qualitatively supports route-level geometric continuity. The reconstructed sequences preserve the main route shape and enough spatial structure for VSLAM in these cases, which complements APE RMSE without implying exact map reconstruction.

\subsection{Payload Size and Runtime Breakdown}
\label{subsec:payload_runtime}

To further analyze transmission behavior, we enable the compression benchmark on a representative 80 s GDUT sequence containing 1921 frames. Table~\ref{tab:payload_runtime} compares raw image ZIP, H.265 MP4 at 10 Mbps, and the VLMem payload. The raw ZIP requires 777.55 MiB, the H.265 reference requires 101.51 MiB, and the VLMem payload requires 15.26 MiB. Thus, on this sequence, the raw ZIP and the 10 Mbps H.265 reference are 51.0$\times$ and 6.7$\times$ larger than the VLMem payload, respectively.

\input{tables/tab_exp4_payload_runtime}

VLMem payload encoding takes 51.80 s, mainly including 40.13 s for phase-correlation motion cues, 10.43 s for semantic event-anchor extraction, and 0.69 s for payload serialization. Cloud-side reconstruction takes 296.64 s, corresponding to 6.59 FPS. Therefore, the current prototype is primarily bottlenecked by cloud-side world-model reconstruction rather than by the compact memory representation. This compute-bound cost depends on the world-model decoder, accelerator memory, and parallel serving strategy; real-time optimized cloud deployment is left for future work.

Table~\ref{tab:payload_runtime} should be read as a comparison of payload size and payload encoding time, not as a comparison of downstream VSLAM accuracy for H.265. On the representative sequence, \VLMemCodec{} shows a strong payload-size reduction relative to raw image ZIP and the 10 Mbps H.265 reference, while the cloud-side reconstruction time remains the dominant compute cost. Communication reduction and reconstruction runtime are therefore separate deployment dimensions: the former is improved by the compact VLMem payload, whereas the latter depends on the world-model backend and serving configuration.

\subsection{Edge-Side Motion Estimation Benchmark}
\label{subsec:motion_benchmark}

The second experiment compares three edge-side motion estimation choices for constructing motion cues. This benchmark evaluates qualitative segmentation consistency and runtime efficiency, rather than supervised motion-label accuracy.

\input{tables/tab_exp2_motion_benchmark}

\begin{figure*}[!t]
\centering
\includegraphics[width=\textwidth]{figures/fig7.png}
\caption{Qualitative comparison of phase correlation, ORB feature tracking, and Farneback optical flow for edge-side motion estimation.}
\label{fig:motion_benchmark}
\end{figure*}

Table~\ref{tab:motion_benchmark} reports the runtime cost of the three edge-side motion estimation choices. The relative cost is normalized by the PC runtime, and the valid ratio denotes the fraction of frame pairs for which a method produced usable motion evidence.

PC is the lightest option, requiring 4.51 s in total and 4.70 ms per frame pair. ORB feature tracking requires 11.73 s, or 2.60x the PC cost, and produces usable motion evidence for 85.7\% of the evaluated pairs. Farneback optical flow (OF) provides a denser motion field, but its runtime increases to 29.75 s, or 6.59x the PC cost.

This comparison reflects the abstraction level required by the encoder. The motion cue does not require pixelwise correspondence or dense local motion; it needs low-cost evidence for coarse motion states and event boundaries that can be converted into language-level motion conditions. ORB is a useful comparison because it is closer to feature-tracking evidence, but it can lose valid output in difficult intervals. OF is useful as a dense-motion reference, but it is substantially more expensive for the edge-side VLMem role.

Fig.~\ref{fig:motion_benchmark} provides the corresponding qualitative evidence. The three temporal color bands indicate the main turning intervals, and PC identifies these event-level turns consistently with ORB and OF in the shown benchmark. PC is therefore not used as a replacement for dense optical flow or feature tracking; it is used as a lightweight cue that is sufficient for coarse, language-level motion conditions in this setting.

The lower-right REC a / REC b comparison in Fig.~\ref{fig:motion_benchmark} further illustrates the role of motion--semantic response fusion. Without motion-estimation guidance, reconstruction can produce an implausible intermediate viewpoint during a right turn. With the full motion--semantic response fusion, the reconstructed sequence shows a smoother and more plausible right-turn transition. This example is qualitative evidence for the encoder design, not a general claim about reconstruction accuracy.

\subsection{MGDR Ablation for the World-Model Decoder}
\label{subsec:lora_ablation}

The final experiment compares the WM and WM+MGDR decoders.

\input{tables/tab_exp3_lora_ablation}

Table~\ref{tab:lora_ablation} reports APE values as RMSE in meters, while LPIPS, SSIM, and FID are auxiliary image-fidelity metrics. The average REC APE decreases from 0.609 m with WM to 0.321 m with WM+MGDR, while the average ORI APE remains 0.240 m because the original sequences are not affected by the decoder. In the terminology of Section~\ref{subsec:wm_lora_decoding}, WM+MGDR denotes the world-model decoder refined through MGDR, implemented with LoRA-based low-rank increments. The REC APE reduction suggests that MGDR biases the decoder toward reconstruction that is more useful for downstream VSLAM in the reported ablation.

The image-fidelity metrics do not improve in the same direction. On average, LPIPS changes from 0.335 to 0.398, SSIM changes from 0.604 to 0.515, and FID changes from 61.286 to 66.552. LPIPS, SSIM, and FID remain useful auxiliary metrics because they characterize perceptual or distribution-level image appearance, but they measure different properties from downstream VSLAM trajectory quality.

Instead, the ablation indicates that traditional image similarity and VSLAM trajectory quality are not fully aligned in this setting. A reconstruction with weaker pixel-level similarity may be more compatible with VSLAM estimation in some cases if it better preserves task-useful geometric continuity, viewpoint stability, and trackable structures. This interpretation is consistent with the task-usable visual reconstruction objective of \VLMemCodec, while the auxiliary image-fidelity metrics still describe important visual tradeoffs.

Fig.~\ref{fig:lora_qualitative} provides representative qualitative evidence for this mismatch. In the shown transitional views, Fig.~\ref{fig:lora_qualitative} qualitatively suggests that WM+MGDR reduces VSLAM-harmful local geometric distortion, implausible reconstructed structures, and hallucinated content compared with WM. The red-circled regions indicate representative WM failures that may damage feature tracking and geometric continuity, which is more consistent with the observed reduction in downstream REC APE than with conventional image-fidelity metrics alone.
