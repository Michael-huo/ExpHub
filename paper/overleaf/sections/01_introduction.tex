\section{Introduction}
\label{sec:introduction}
% ==================================================

\IEEEPARstart{V}{isual} simultaneous localization and mapping (VSLAM) enables mobile robots, autonomous platforms, and embodied intelligent systems to estimate motion and build maps from visual observations. With the increasing availability of cloud-side computation, edge--cloud VSLAM has become an attractive paradigm: the edge device captures visual observations with limited onboard resources, while the cloud executes more powerful perception, optimization, and mapping modules. This division, however, shifts a critical burden to the communication link. A monocular or RGB-D camera continuously produces high-frame-rate image streams, and transmitting dense visual data from the edge to the cloud incurs substantial bandwidth, energy, and latency costs. The bottleneck becomes more severe in long-term navigation, large-scale deployment, and bandwidth-constrained wireless environments.

Existing image and video compression standards reduce this burden by exploiting spatial and temporal pixel redundancy. Such codecs are highly effective for human viewing, and their rate--distortion objectives are mainly designed around visual fidelity, perceptual quality, and compact delivery of visually faithful frames. These objectives are not explicitly VSLAM-oriented. A compressed stream may spend bits preserving fine appearance details, color variations, or human-perceptual textures that do not necessarily provide stable trackable structures for camera pose estimation.

The mismatch becomes more important when higher compression ratios are pursued. Distortions such as blur, blocking artifacts, texture loss, or temporal inconsistency may weaken feature repeatability, disturb trackable structures, and reduce the geometric cues used by downstream pose estimation. Conversely, VSLAM does not require pixel-identical or high-definition reconstruction of every frame; it requires sufficient geometric consistency, feature repeatability, trackable structures, and scene-motion continuity. Therefore, edge--cloud VSLAM transmission should not be treated merely as conventional video delivery, but as a VSLAM-oriented visual transmission problem in which the transmitted representation is formulated and evaluated by downstream VSLAM trajectory quality. Fig.~\ref{fig:concept} conceptually illustrates this motivation, while the quantitative compression and trajectory tradeoffs are reported in Section~\ref{sec:experiments}.

As a design analogy, this paper is motivated by a human-inspired view of visual memory. When navigating through an environment, humans need not retain every pixel of every frame; they often recall sparse key scenes, landmarks, motion transitions, and high-level semantic impressions. This analogy does not provide a biological model, but it suggests a compact representation for edge--cloud VSLAM visual transmission: a dense visual stream can be represented by sparse vision--language memory. In this representation, boundary frames provide geometric anchors, motion transitions and semantic anchors indicate what should be retained, language memory provides compact high-level conditions, and a cloud-side world-model decoder reconstructs task-usable visual continuity.

The key intuition is that many pixel-level details can often be omitted when task-useful visual structures are preserved. HVM-Codec is designed to discard more VSLAM-irrelevant visual redundancy while preserving such structures through sparse boundary frames, motion memory, semantic anchors, and language memory. This differs from simple keyframe transmission because the omitted intervals are reconstructed by the world-model decoder under vision--language conditions, rather than being left as gaps between transmitted frames.

Based on this formulation, we propose \emph{HVM-Codec}, a human-inspired vision--language memory coding framework for edge--cloud VSLAM. For each vision--language memory unit, the edge transmits boundary frames as geometric conditions and encodes compact language memory to describe motion intent, scene priors, and reconstruction conditions. The boundary frames constrain the start and end appearance of the segment, while the language memory represents part of the omitted visual information with a compact abstraction. The cloud-side vision--language conditioned world-model decoder uses its learned visual prior to reconstruct task-usable intermediate frames. The reconstructed sequence is then fed into a cloud-side VSLAM estimator for trajectory estimation.

The key mechanism behind HVM-Codec is the combination of three complementary information sources. First, boundary frames preserve visual and geometric conditions that are difficult to describe purely by language. Second, language memory encodes high-level motion and semantic priors with low communication cost. Third, the cloud-side world-model decoder supplies plausible intermediate visual continuity from its learned prior under the vision--language conditions. In this way, dense visual transmission is reformulated as a compact vision--language memory transmission problem rather than a generic video compression problem.

To make this framework practical for edge--cloud VSLAM, HVM-Codec adopts a lightweight edge-side memory encoder. Instead of running heavy online captioning or dense optical-flow networks on the edge, the encoder uses low-cost visual measurements to estimate coarse motion memory and semantic anchors. These cues are then converted into structured language memory through a language vocabulary and motion-conditioned language templates. In the cloud decoding stage, memory-guided world-model refinement, implemented with Low-Rank Adaptation (LoRA), biases the decoder toward navigation-style visual transitions and VSLAM-oriented reconstruction. In this work, reconstruction is evaluated primarily by downstream VSLAM trajectory quality using absolute pose error (APE) RMSE, while transmission reduction, edge-side motion-estimation cost, and auxiliary image-fidelity metrics are reported to characterize the tradeoffs.

The main contributions of this paper are summarized as follows:
\begin{itemize}
    \item We propose a human-inspired vision--language memory coding framework for edge--cloud VSLAM that replaces dense frame transmission with sparse boundary frames and compact language memory.
    \item We design a lightweight edge-side vision--language memory encoder that selects boundary frames, extracts coarse motion memory and semantic-anchor cues, and converts them into compact language memory without relying on heavy online generative captioning.
    \item We develop a vision--language conditioned cloud-side world-model decoder with memory-guided refinement implemented with LoRA, where boundary frames and language memory jointly constrain task-usable intermediate-frame reconstruction.
    \item We evaluate VSLAM-oriented visual transmission tradeoffs through compression and APE RMSE results, edge-side motion-estimation cost, and WM/WM+LoRA behavior with auxiliary LPIPS, SSIM, and FID metrics.
\end{itemize}
