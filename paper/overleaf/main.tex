% -------------------- Abstract --------------------
\begin{abstract}
Edge--cloud visual simultaneous localization and mapping (VSLAM) relies on continuous visual streams, but transmitting dense image sequences from resource-constrained edge devices to cloud-side SLAM backends introduces substantial bandwidth and energy costs. Existing image and video codecs mainly optimize pixel-level fidelity or perceptual quality, which does not necessarily preserve the geometric continuity and trackable structures required by VSLAM. Inspired by human visual memory, this paper proposes \HVM, a human-inspired image--text visual memory coding framework for edge--cloud VSLAM. The edge encoder represents a visual sequence using boundary frames, lightweight motion memories, visual novelty anchors, and compact language conditions. Boundary frames provide geometric constraints, language memories encode high-level motion and semantic priors, and a cloud-side world-model decoder uses its learned visual prior to imagine and reconstruct task-usable intermediate frames. To improve SLAM-friendly reconstruction, the decoder is further fine-tuned with LoRA. The proposed codec is evaluated by downstream VSLAM trajectory quality rather than pixel fidelity alone, including absolute pose error, relative pose error, trajectory completeness, bandwidth reduction, and runtime.
\end{abstract}

\section{Introduction}
\label{sec:introduction}
% ==================================================

\IEEEPARstart{V}{isual} simultaneous localization and mapping (VSLAM) is a fundamental capability for mobile robots, autonomous platforms, and embodied intelligent systems. With the increasing availability of cloud-side computation, edge--cloud VSLAM has become an attractive paradigm: the edge device captures visual observations with limited onboard resources, while the cloud can execute more powerful perception, optimization, and mapping modules. However, this paradigm introduces a critical communication bottleneck. A monocular or RGB-D camera continuously produces high-frame-rate image streams, and transmitting dense visual data from the edge to the cloud incurs substantial bandwidth, energy, and latency costs. This bottleneck becomes more severe in long-term navigation, large-scale deployment, and bandwidth-constrained wireless environments.

Existing image and video compression standards reduce visual data by exploiting spatial and temporal pixel redundancy. Although such codecs are highly effective for human viewing, their objectives are not directly aligned with VSLAM. VSLAM does not require every pixel to be faithfully reconstructed; instead, it relies on task-usable geometric continuity, stable local structures, trackable visual textures, and temporally consistent camera motion. Under aggressive compression, images that remain visually acceptable to humans may lose feature repeatability, introduce temporal artifacts, or distort geometric cues that are essential for downstream pose estimation. Therefore, edge--cloud VSLAM transmission should not be treated merely as conventional video delivery, but as a task-oriented coding problem in which the transmitted representation is optimized for downstream localization and mapping.

This paper is motivated by a human-inspired view of visual memory. Humans do not memorize every pixel of every frame when navigating through an environment. Instead, they tend to retain sparse key scenes, salient landmarks, motion transitions, and high-level semantic impressions, while reconstructing the continuous visual experience from memory and prior knowledge. For example, when recalling a route, one may remember the entrance of a corridor, a left turn, a distinctive object, and the final destination, rather than all intermediate images. This observation suggests a different formulation for edge--cloud VSLAM image transmission: a dense visual stream can be compressed into sparse image-text visual memories, where images provide geometric anchors, language provides compact high-level information, and a cloud-side generative model reconstructs the missing visual continuity.

Based on this idea, we propose \emph{HVM-Codec}, a human-inspired image-text visual memory coding framework for edge--cloud VSLAM. For each visual memory unit, the edge transmits boundary frames as geometric constraints and constructs compact language memories to describe motion intent, scene priors, and reconstruction constraints. The boundary frames constrain the start and end appearance of the segment, while the language memory represents part of the missing visual information with a much lower-bit abstraction. On the cloud side, an image-text conditioned world-model decoder uses its learned visual prior to reconstruct task-usable intermediate frames. The reconstructed sequence is then fed into a cloud-side VSLAM backend for trajectory estimation.

The key mechanism behind HVM-Codec is the combination of three complementary information sources. First, boundary frames preserve visual and geometric constraints that are difficult to describe purely by language. Second, language memories encode high-level motion and semantic priors with extremely small communication cost. Third, the cloud-side world model supplies missing low-level visual details from its learned prior under the image-text constraints. In this way, dense visual transmission is reformulated as a compact image-text memory transmission problem.

To make this framework practical for edge--cloud VSLAM, HVM-Codec adopts a lightweight edge-side memory encoder. Instead of running heavy online captioning or dense optical-flow networks on the edge, the encoder extracts coarse motion memories and visual novelty cues using low-cost visual measurements. These cues are converted into structured language conditions through a controllable prompt vocabulary and motion-conditioned templates. The cloud-side decoder is further fine-tuned with Low-Rank Adaptation (LoRA), which biases the world model toward navigation-style visual transitions and SLAM-friendly reconstruction. In this work, the quality of reconstruction is evaluated not only by visual appearance, but more importantly by downstream VSLAM trajectory quality, including absolute pose error, relative pose error, trajectory completeness, tracking success, compression ratio, and runtime.

The main contributions of this paper are summarized as follows:
\begin{itemize}
    \item We propose a human-inspired image-text visual memory coding framework for edge--cloud VSLAM, which replaces dense frame transmission with sparse boundary frames and compact language memories.
    \item We design a lightweight edge-side visual memory encoder that extracts motion memories and visual novelty cues, and converts them into compact language conditions without relying on heavy online generative captioning.
    \item We develop an image-text conditioned cloud-side world-model decoder, where boundary frames provide geometric constraints, language memories provide high-level semantic and motion priors, and LoRA fine-tuning improves SLAM-friendly reconstruction.
    \item We establish a task-level evaluation protocol for VSLAM-oriented visual transmission, measuring compression performance by downstream trajectory quality, trajectory completeness, tracking robustness, bandwidth reduction, and runtime rather than pixel fidelity alone.
\end{itemize}

The remainder of this paper is organized as follows. Section~\ref{sec:related_work} reviews related work on edge--cloud VSLAM transmission, human-inspired visual memory, and world-model-based generative reconstruction. Section~\ref{sec:methodology} presents the proposed HVM-Codec framework. Section~\ref{sec:experiments} reports the experimental evaluation and ablation studies. Section~\ref{sec:discussion} discusses the characteristics and limitations of the proposed method. Section~\ref{sec:conclusion} concludes the paper.

\section{Related Work}
\label{sec:related_work}
% ==================================================


\subsection{Edge--Cloud VSLAM and Task-Oriented Visual Transmission}
\label{subsec:edge_cloud_transmission}

Visual SLAM has evolved from feature-based and direct pipelines to learning-based systems with stronger robustness. ORB-SLAM2 and ORB-SLAM3 are representative feature-based systems that build sparse maps from repeatable local features and optimize camera poses through bundle adjustment~\cite{murartal2017orbslam2,campos2021orbslam3}. Direct and semi-direct methods such as LSD-SLAM, DSO, and SVO exploit photometric consistency to reduce descriptor dependence~\cite{engel2014lsdslam,engel2018dso,forster2014svo}, while learning-based systems such as DROID-SLAM introduce dense recurrent optimization and achieve strong performance across monocular, stereo, and RGB-D inputs~\cite{teed2021droidslam}. These advances improve localization and mapping capability, but they also increase computational demand, motivating edge--cloud collaboration for resource-constrained robots.

Cloud-edge SLAM moves expensive mapping, optimization, or reconstruction modules from the robot to the cloud. Existing systems and reviews show that transmission latency and data volume are central bottlenecks, and identify asynchronous collaboration, submap-level allocation, and compact representation transmission as important directions~\cite{rabaud2011c2tam,kuffner2010cloud,shankar2020fogros,chen2024reviewcloudedge}. Recent submap-based cloud-edge VSLAM reduces transmission frequency by offloading challenging image segments or submap construction to the cloud~\cite{chen2024cloudlearning,chen2024implicittransmission}. Implicit-representation transmission and VC-SLAM further reduce transmitted map data by sending compact neural representations instead of dense point clouds or meshes~\cite{chen2024implicittransmission,huo2025vcslam}, while X-RepSLAM shows that visual sequence regions can be assigned to different representation models according to information richness~\cite{chen2025xrepslam,chen2024crossdata}.

These works establish the importance of reducing transmission demand in cloud-edge VSLAM. However, they mainly study task allocation, submap transmission, map representation, or data association after visual observations have been selected. HVM-Codec focuses on the preceding visual transmission problem: how to encode the edge image stream itself into a compact representation before cloud-side VSLAM. This goal is related to task-oriented compression and semantic communication. Conventional codecs such as H.264/AVC, H.265/HEVC, and H.266/VVC optimize rate--distortion objectives designed mainly for human viewing~\cite{sullivan2012h264,sullivan2012hevc,bross2021vvc}. Learned and generative compression methods exploit neural priors to reconstruct plausible details under limited bit budgets~\cite{balle2018variational,minnen2018joint,agustsson2020scale,mentzer2020highfidelity}, and semantic communication argues that task-relevant meaning should be transmitted instead of raw symbols~\cite{qin2021semantic,luo2022semantic}. In contrast to these general formulations, HVM-Codec evaluates the transmitted representation by downstream VSLAM trajectory quality.
\subsection{Human-Inspired Visual Memory and Event-Based Representation}
\label{subsec:visual_memory_event}

HVM-Codec is motivated by the observation that visual navigation does not require remembering every pixel of every frame. Cognitive studies on event perception suggest that humans segment continuous experience into discrete events, and that event boundaries are associated with changes in prediction, attention, and memory~\cite{zacks2001event,zacks2007event}. Mental-model theory further suggests that humans reason by constructing simplified internal representations that preserve structural relationships rather than exhaustive sensory details~\cite{johnsonlaird1983mental}. For navigation, this implies that sparse key scenes, landmarks, and motion transitions can serve as compact memory cues for reconstructing a route-level experience.

Computer vision has studied related ideas through temporal segmentation, change-point detection, and event boundary detection. Generic Event Boundary Detection (GEBD) detects taxonomy-free boundaries in videos according to human-perceived event changes~\cite{shou2021gebd}. Online Bayesian change-point detection provides a statistical foundation for streaming segmentation~\cite{adams2007bayesian}, and recent representation-learning methods study how streaming visual inputs can be organized into event-level structures~\cite{streamer2023}. Vision-language representation models, such as CLIP and BLIP-2, further provide a way to connect visual content with compact language-level semantics~\cite{radford2021clip,li2023blip2}. In VSLAM, X-RepSLAM shows that VLM-derived image-sequence information can guide adaptive representation switching and prompt-driven task customization~\cite{chen2025xrepslam}.

Our work adopts this memory-oriented view for edge--cloud VSLAM transmission. Motion changes and visual novelty are used not only to segment a sequence, but also to construct a compact image--text memory representation. Boundary frames preserve geometric constraints; motion states and visual anchors determine what should be remembered; and language memories encode high-level motion and semantic priors with very low data volume. The selected cues are organized to support cloud-side world-model reconstruction and downstream VSLAM.

\subsection{World Models and Generative Visual Reconstruction}
\label{subsec:world_model_decoding}

World models provide the conceptual basis for reconstructing missing visual experience from sparse image--text cues. Ha and Schmidhuber revived the term ``world model'' by learning latent environment dynamics for agents that can simulate possible futures~\cite{ha2018worldmodels}. The Dreamer family further demonstrates that latent world models can support decision-making and control from imagined rollouts~\cite{hafner2019planet,hafner2020dreamer,hafner2023dreamerv3}. JEPA-style predictive representation learning emphasizes predicting abstract latent representations instead of raw pixels, which is consistent with the idea that useful world knowledge should filter redundant sensory details~\cite{lecun2022path,assran2024vjepa}. Recent surveys categorize world models into two broad functions: constructing internal representations to understand the external world and predicting future states to support simulation and decision-making~\cite{ding2025worldmodelsurvey}.

Large-scale video generation models have pushed world models toward visual imagination and future prediction. Image-to-video and text-to-video diffusion models can synthesize temporally coherent sequences from sparse visual or language conditions~\cite{blattmann2023svd,ho2022videodiffusion,villegas2022phenaki}. Reports on Sora, Genie, Cosmos, and related systems argue that large video models are increasingly able to capture spatiotemporal consistency, controllability, and physically plausible visual transitions~\cite{brooks2024sora,bruce2024genie,nvidia2025cosmos}. In robotics and embodied AI, world models have been explored for learning from imagined futures, generating robot action videos, and supporting planning~\cite{yang2023unipi,du2023learning,wu2023daydreamer}. Target-Bench is especially relevant: it evaluates whether world models can generate future videos for mapless path planning toward semantic targets, and uses a world decoder to recover camera motion from generated videos and compare it with SLAM-based ground truth trajectories~\cite{wang2025targetbench}.

Although these works show that world models can generate plausible future visual states, directly using a generic generator for VSLAM-oriented reconstruction is insufficient. VSLAM requires stable viewpoints, geometrically consistent scene structures, and trackable textures, while general video models are typically optimized for visual realism, diversity, or prompt adherence. Efficient low-rank fine-tuning methods such as LoRA make it possible to bias a pretrained generator toward a target domain without retraining the full model~\cite{hu2022lora}. HVM-Codec therefore treats the cloud decoder as an image--text conditioned world model and fine-tunes it toward navigation-style visual transitions. The reconstructed frames are not expected to be pixel-identical to the original frames; instead, they are evaluated by whether they preserve enough visual continuity for downstream VSLAM trajectory estimation.

\section{Methodology}

\subsection{Framework Overview and Problem Formulation}
\label{subsec:framework_formulation}

\HVM formulates edge--cloud VSLAM image transmission as an image--text visual memory coding problem. Given a dense edge image stream, the encoder converts it into a compact memory representation consisting of boundary frames, event-based visual memory units, language memories, and generation constraints. The cloud-side world-model decoder then reconstructs task-usable intermediate frames from the transmitted image--text memory, and the reconstructed sequence is used by a cloud VSLAM backend.

\begin{figure*}[!t]
\centering
\placeholderfig{1.75in}{0.92\textwidth}
\caption{Overall framework of \HVM. The edge-side memory encoder converts a dense image stream into boundary frames, motion-semantic event units, and compact language memories. The same encoder is also reused to construct LoRA fine-tuning pairs for the world-model decoder. During inference, sparse image--text memories are transmitted to the cloud, where the world-model decoder reconstructs intermediate frames for downstream VSLAM.}
\label{fig:framework}
\end{figure*}

Let the input visual stream be
\begin{equation}
\Iseq=\{I_1,I_2,\ldots,I_T\}.
\end{equation}
The edge encoder produces a set of visual memory units
\begin{equation}
\mathcal{U}=\{U_1,U_2,\ldots,U_N\},
\end{equation}
where each unit is represented by
\begin{equation}
U_i=\{I_i^s,I_i^e,S_i,A_i,L_i,L_i^-\}.
\label{eq:memory_unit}
\end{equation}
Here \(I_i^s\) and \(I_i^e\) denote the start and end boundary frames, \(S_i\) denotes the motion memory, \(A_i\) denotes visual or semantic anchors, and \(L_i\) and \(L_i^-\) denote positive and negative language memories. The transmitted memory can be written as
\begin{equation}
\Mmem=\{\mathcal{U},\Pprompt,\mathrm{metadata}\},
\end{equation}
where \(\Pprompt\) denotes the prompt vocabulary and generation constraints shared by the encoder and decoder.

The cloud-side decoder reconstructs the visual sequence unit by unit:
\begin{equation}
\hat{\mathcal{I}}_i=G_{\theta+\Delta\theta}(I_i^s,I_i^e,L_i,L_i^-),
\label{eq:wm_decode}
\end{equation}
where \(G_\theta\) is a pretrained world model and \(\Delta\theta\) denotes the low-rank increments learned by LoRA fine-tuning. The reconstructed sequence is obtained by temporal merging:
\begin{equation}
\Ihat=\operatorname{Merge}(\hat{\mathcal{I}}_1,\hat{\mathcal{I}}_2,\ldots,\hat{\mathcal{I}}_N).
\end{equation}

The transmission budget is
\begin{equation}
B(\Mmem)=B(\Kframes)+B(\Llang)+B(\Pprompt)+B(\mathrm{metadata}),
\label{eq:budget}
\end{equation}
where \(B(\Llang)\ll B(\Iseq_{\mathrm{missing}})\). The coding objective is therefore formulated as
\begin{equation}
\min B(\Mmem),\quad \mathrm{s.t.}\quad \Delta_{\mathrm{VSLAM}}(\Ihat,\Iseq)\leq \epsilon,
\label{eq:objective}
\end{equation}
where \(\Delta_{\mathrm{VSLAM}}\) measures downstream trajectory degradation using APE, RPE, trajectory completeness, and tracking success.

An important feature of the framework is that the same encoder is used in both deployment and training. During inference, the encoder produces the memory units transmitted to the cloud. During offline training, the encoder converts raw training sequences into image--text pairs for LoRA fine-tuning:
\begin{equation}
\mathcal{D}_{\mathrm{LoRA}}
=\{(I_i^s,I_i^e,L_i,L_i^-,I_i^{s+1:e-1})\}_{i=1}^{N_d},
\label{eq:lora_dataset}
\end{equation}
where \(I_i^{s+1:e-1}\) denotes the original intermediate frames. This shared encoder design aligns the memory format used for world-model fine-tuning with the memory format used during inference.

\subsection{Motion-Semantic Event Segmentation for Boundary-Frame Selection}
\label{subsec:motion_semantic_segmentation}

The first function of the edge encoder is to determine which images should be transmitted as boundary frames. HVM-Codec treats this as a motion-semantic event segmentation problem in the first-person visual stream. A useful boundary should capture a transition that is important for reconstructing camera motion or scene content. Therefore, the segmentation is driven by two complementary cues: lightweight motion response and visual semantic novelty.

For consecutive frames, a global motion response is estimated using phase correlation:
\begin{equation}
(\Delta x_t,\Delta y_t,\rho_t)=\operatorname{PC}(I_{t-1},I_t),
\end{equation}
where \(\Delta x_t\) and \(\Delta y_t\) describe the dominant translational response and \(\rho_t\) denotes the response confidence. A coarse first-person motion state is then assigned as
\begin{equation}
s_t\in\{\mathrm{forward},\mathrm{left\mbox{-}turn},\mathrm{right\mbox{-}turn},\mathrm{mixed}\}.
\label{eq:motion_state}
\end{equation}
The accumulated motion change within a candidate interval \([t_0,t]\) is denoted by
\begin{equation}
R_m(t_0,t)=\sum_{k=t_0+1}^{t}\phi_m(s_{k-1},s_k,\rho_k),
\label{eq:motion_event_score}
\end{equation}
where \(\phi_m(\cdot)\) measures state transition strength and confidence-weighted motion variation.

Motion alone is insufficient when the camera maintains a stable forward or turning state for a long period. In such cases, the motion label may remain unchanged while the scene content changes significantly. To capture these changes, the encoder also measures visual novelty in an embedding space. Given an image encoder \(f(\cdot)\), the frame-level novelty is
\begin{equation}
d_t=1-\cos(f(I_t),f(I_{t-1})),
\label{eq:visual_novelty}
\end{equation}
and the accumulated visual novelty is
\begin{equation}
R_v(t_0,t)=\sum_{k=t_0+1}^{t}d_k.
\label{eq:visual_event_score}
\end{equation}

The event boundary set is obtained by combining the two responses:
\begin{equation}
\mathcal{B}=\Pi_{\Omega}\left(\left\{t\mid R_m(t_0,t)>\tau_m\ \vee\ R_v(t_0,t)>\tau_v\right\}\right),
\label{eq:event_boundary}
\end{equation}
where \(\tau_m\) and \(\tau_v\) are motion and visual novelty thresholds, and \(\Pi_{\Omega}(\cdot)\) projects detected boundaries to feasible temporal positions. The boundary set \(\mathcal{B}\) defines the visual memory units. For the \(i\)-th unit, its motion memory and visual anchor set are
\begin{equation}
S_i=\operatorname{Merge}(s_t,\ t\in[t_i^s,t_i^e]),
\end{equation}
\begin{equation}
A_i=\{t\in[t_i^s,t_i^e]\mid d_t>\tau_a\ \mathrm{or}\ R_v(t_i^s,t)>\tau_v\}.
\end{equation}

\subsection{Controlled Language Memory Coding for Non-Transmitted Frames}
\label{subsec:language_memory}

The second function of the edge encoder is to represent non-transmitted visual information using compact language memory. For each unit, the boundary frames specify visual and geometric endpoints, while language memory provides high-level information about how the missing visual segment should evolve. This language memory is constructed from a controlled vocabulary rather than online free-form caption generation.

The positive language memory of unit \(i\) is defined as
\begin{equation}
L_i=L_{\mathrm{base}}\oplus L_{\mathrm{motion}}(S_i)\oplus L_{\mathrm{anchor}}(A_i)\oplus L_{\mathrm{constraint}},
\label{eq:language_memory}
\end{equation}
where \(L_{\mathrm{base}}\) encodes a first-person navigation-view prior, \(L_{\mathrm{motion}}(S_i)\) describes the camera motion state, \(L_{\mathrm{anchor}}(A_i)\) optionally inserts visual-anchor descriptions, and \(L_{\mathrm{constraint}}\) specifies temporal and geometric reconstruction requirements. The negative language memory is
\begin{equation}
L_i^- = L_{\mathrm{blur}}\oplus L_{\mathrm{distortion}}\oplus L_{\mathrm{drift}}\oplus L_{\mathrm{jump}}\oplus L_{\mathrm{hallucination}},
\label{eq:negative_language}
\end{equation}
which suppresses visual artifacts that are harmful to VSLAM, such as blur, geometric distortion, temporal drift, sudden viewpoint jumps, and inconsistent hallucinated objects.

The motion-conditioned part is generated through a deterministic mapping
\begin{equation}
L_{\mathrm{motion}}(S_i)=\Gamma(S_i),
\label{eq:motion_language_mapping}
\end{equation}
where \(\Gamma(\cdot)\) maps motion states such as forward motion, left turn, and right turn into short textual descriptions. The resulting memory size is negligible compared with the omitted image sequence:
\begin{equation}
B(L_i)+B(L_i^-)\ll B(I_i^{s+1:e-1}).
\label{eq:language_compression}
\end{equation}

\subsection{World-Model Decoding and LoRA Fine-Tuning for VSLAM-Friendly Reconstruction}
\label{subsec:wm_lora_decoding}

On the cloud side, the transmitted image--text memory is decoded by a world model. Unlike a conventional bitstream decoder, the world-model decoder reconstructs missing frames by combining transmitted boundary images, compact language memory, and learned spatiotemporal priors. For each unit, decoding is performed as in (\ref{eq:wm_decode}), and the generated intermediate frames are temporally merged to form the sequence used by the VSLAM backend.

The boundary frames and language memories play complementary roles. The frames \(I_i^s\) and \(I_i^e\) constrain the start and end appearance, camera viewpoint, and local trackable structures. The positive language memory \(L_i\) provides motion and semantic priors, while the negative language memory \(L_i^-\) constrains visual artifacts. The world model then supplies plausible low-level details and intermediate viewpoint changes from its learned prior.

To bias the decoder toward navigation-style visual transitions, the pretrained world model is fine-tuned with LoRA. Let \(G_\theta\) be the base world model. LoRA introduces low-rank increments \(\Delta\theta\) to selected layers, leading to the tuned decoder \(G_{\theta+\Delta\theta}\). The fine-tuning data are generated by the same encoder according to (\ref{eq:lora_dataset}), which provides boundary frames, language memories, and the corresponding original intermediate frames. A reconstruction-oriented training objective can be written as
\begin{equation}
\Delta\theta^*=\arg\min_{\Delta\theta}\sum_i
\mathcal{L}_{\mathrm{gen}}\left(G_{\theta+\Delta\theta}(I_i^s,I_i^e,L_i,L_i^-), I_i^{s+1:e-1}\right),
\label{eq:lora_finetune}
\end{equation}
where \(\mathcal{L}_{\mathrm{gen}}\) denotes the generation training objective used by the video world model. The purpose of this fine-tuning is to align the world model with the memory format used by HVM-Codec and to improve viewpoint stability, geometric continuity, and feature trackability for downstream VSLAM.

\section{Discussion}

\HVM differs from traditional video compression because it optimizes for downstream VSLAM utility rather than pixel fidelity alone. Compact language memory represents part of the missing visual information and activates the cloud-side world prior under boundary-frame constraints.

\section{Conclusion}
\label{sec:conclusion}
% ==================================================

This paper presented \HVM, a human-inspired image--text visual memory coding framework for edge--cloud VSLAM. The proposed formulation compresses dense image streams into boundary frames, motion memories, visual anchors, and compact language conditions, and reconstructs task-usable visual sequences on the cloud using an image--text conditioned world-model decoder with LoRA fine-tuning. Future experiments will validate the compression--trajectory trade-off across multiple VSLAM datasets and real robot scenes.
